Aplikasi Kantin Digital "Hulahup" (Food-TYU) Telkom University

📌 Pendahuluan & Ringkasan Sistem

Hulahup App (Food-TYU) adalah sistem informasi e-canteen berbasis web yang dibangun untuk mengatasi masalah antrean panjang dan kerumitan transaksi tunai di kantin kampus Telkom University. Aplikasi ini menerapkan konsep Single Checkout & Split Order Processing, di mana pembeli dapat memesan makanan dari kantin berbeda sekaligus, namun diproses secara terpisah di masing-masing dapur kantin.

Aplikasi ini memiliki keunggulan integrasi gerbang pembayaran online Midtrans untuk kemudahan pengisian saldo digital, serta asisten kuliner pintar menggunakan teknologi generative Google Gemini AI API.

🛠️ Bedah Kode & Cara Pembuatan Fitur (Teknis)

Gunakan penjelasan ini jika dosen bertanya **"bagaimana cara kamu membuat fitur ini?"** atau meminta Anda menjelaskan potongan kodenya:

🎨 1. Bagaimana Cara Membuat Animasi Bergerak di Halaman?

Semua animasi visual yang berjalan di landing page (seperti kartu melayang, teks berjalan, dan cincin notifikasi) dibangun menggunakan kombinasi CSS Keyframes dan Tailwind Utility Classes.

A. Animasi Kartu/Hero Melayang (Floating Effect)
Untuk memberikan kesan modern dan premium, gambar hero melayang naik turun secara halus menggunakan interpolasi transform pada sumbu Y.

/* Didefinisikan di resources/views/welcome.blade.php */
@keyframes floatY {
    0%, 100% { transform: translateY(0); }
    50% { transform: translateY(-12px); }
}

.animate-float-slow {
    animation: floatY 6s ease-in-out infinite;
}

B. Animasi Teks & Logo Berjalan (Infinite Marquee)
Banner nama-nama kantin bergerak tanpa henti secara horizontal. Kami menggunakan transform: translateX dari 0% ke -50% dengan wrapper yang diduplikasi datanya sehingga pergeserannya terlihat tidak terputus (*seamless loop*).

@keyframes marquee {
    from { transform: translateX(0); }
    to { transform: translateX(-50%); }
}

.animate-marquee {
    display: flex;
    width: max-content;
    animation: marquee 20s linear infinite;
}

C. Animasi Cincin Notifikasi Chatbot (Pulse Ring)
Untuk memancing interaksi pengguna ke arah chatbot tanpa mengganggu estetika halaman, kami menggunakan efek gelombang berdenyut di sekitar ikon robot chatbot.

@keyframes pulseRing {
    0% { transform: scale(0.95); box-shadow: 0 0 0 0 rgba(2, 132, 199, 0.5); }
    70% { transform: scale(1); box-shadow: 0 0 0 10px rgba(2, 132, 199, 0); }
    100% { transform: scale(0.95); box-shadow: 0 0 0 0 rgba(2, 132, 199, 0); }
}

Poin Tambahan Sidang: Penulisan animasi dilakukan dengan menargetkan properti transform dan opacity karena kedua properti tersebut didelegasikan langsung ke GPU (*Hardware Acceleration*), sehingga animasi tetap berjalan lancar pada kecepatan 60 FPS tanpa membebani memori CPU browser.

🤖 2. Bagaimana Cara Membuat Fitur AI Chatbot (Gemini AI)?

Fitur Chatbot dibangun dengan arsitektur Client-Server API Integration yang menghubungkan frontend web dengan Google Gemini API melalui backend Laravel secara aman.

A. Sisi Frontend (Vanilla JavaScript di View layout)

Saat pengguna mengetik pesan dan mengkliknya, JavaScript akan mengambil riwayat percakapan (*chat history*) serta pesan baru, kemudian mengirimkannya via request POST menggunakan API fetch():

// Mengirim pesan secara asynchronous
const response = await fetch('/api/chatbot/chat', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').getAttribute('content')
    },
    body: JSON.stringify({
        message: userTextInput,
        history: chatHistoryArray // Riwayat dikirim agar chatbot ingat konteks obrolan sebelumnya
    })
});
const data = await response.json();
appendBotReply(data.reply); // Tampilkan balasan chatbot

B. Sisi Backend (ChatbotController.php)

Di backend, Laravel melakukan pengumpulan data menu secara dinamis dan menembak REST API Google Gemini:

  1. Mengambil Menu Aktif dari Database SQL:
    $menus = Menu::where('is_available', true)->with('canteen')->get();
    Data menu ini diubah menjadi format teks daftar terstruktur.
  2. Menyusun System Instruction (Instruksi Dasar AI):
    Kami menyuntikkan data menu nyata tersebut ke dalam perintah sistem, serta mendikte gaya bahasanya agar menggunakan slang kampus Tel-U (menggunakan kata ganti *kamu*, *kak*, *bro*, dll.) serta menginstruksikan AI untuk hanya merekomendasikan menu yang ada dalam daftar tersebut.
  3. Menembak Endpoint Google Gemini API via HTTP Client Laravel:
// Model yang digunakan: gemini-2.5-flash (terbaru, cepat, hemat kuota token)
$response = Http::timeout(15)->post(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=" . env('GEMINI_API_KEY'), 
    [
        'contents' => $contents, // Gabungan history chat dan pesan user baru
        'systemInstruction' => [
            'parts' => [
                ['text' => $systemInstruction] // Berisi data database menu kampus Tel-U
            ]
        ]
    ]
);

if ($response->successful()) {
    $reply = $response->json()['candidates'][0]['content']['parts'][0]['text'];
    return response()->json(['reply' => $reply]);
}

Poin Tambahan Sidang: Penggunaan env('GEMINI_API_KEY') menjaga keamanan kredensial agar API key tidak bocor ke publik atau repositori GitHub. Jika koneksi API gagal, sistem dilindungi oleh blok try-catch yang mengembalikan rekomendasi cadangan (*fallback*) tanpa merusak antarmuka pengguna.

🧭 Alur Skenario Demo Presentasi (Step-by-Step)

Gunakan urutan langkah berikut ini saat mendemokan aplikasi di depan dosen penguji agar presentasi berjalan sistematis dan menonjolkan fitur-fitur utama:

1

Landing Page Utama & Informasi Realtime

Aksi Demo: Buka halaman depan http://localhost:8000/ sebelum melakukan login. Tunjukkan bagian statistik di Hero section.
Poin Penjelasan untuk Dosen:
  • Jelaskan bahwa data statistik ("Kantin Aktif", "Pilihan Menu") ditarik secara realtime dari database melalui relasi model, bukan angka hardcode.
  • Tunjukkan baris Live Order Ticker (pesanan terbaru pembeli) yang bergerak secara dinamis sesuai transaksi terakhir di database.
  • Tunjukkan section "Menu Terpopuler" yang diurutkan otomatis berdasarkan menu yang paling banyak dibeli pada transaksi sukses.
2

Registrasi Akun Mahasiswa (Otomasi RBAC)

Aksi Demo: Klik tombol Sign Up. Pilih tipe akun Mahasiswa Tel-U. Masukkan email berdomain @student.telkomuniversity.ac.id dan isi NIM (12 digit). Klik daftar dan login.
Poin Penjelasan untuk Dosen:
  • Jelaskan sistem penentuan peran otomatis di AuthController.php. Jika pembeli mendaftar dengan NIM atau email Tel-U, sistem secara otomatis memberi peran mahasiswa (bukan user biasa).
  • Mahasiswa Tel-U secara otomatis mendapatkan bonus saldo awal e-wallet sebesar **Rp 50.000** (diatur di database seeder/registrasi) untuk mensimulasikan sistem e-money internal kampus.
  • Tunjukkan lencana khusus "✨ Mahasiswa Tel-U" yang muncul di navbar pembeli setelah login.
3

Keranjang Pemesanan Multi-Kantin

Aksi Demo: Masuk ke halaman Kantin. Pilih Kantin Barokah, lalu masukkan Bakso Malang ke keranjang. Kembali, pilih Kantin Segar, lalu masukkan Bubble Ice Taro. Buka halaman keranjang belanja (Cart).
Poin Penjelasan untuk Dosen:
  • Tunjukkan bahwa keranjang belanja secara cerdas **mengelompokkan menu berdasarkan kantin asalnya** secara visual.
  • Jelaskan konsep *Split Order* di backend. Saat pembeli mengklik checkout, pesanan akan dipecah menjadi beberapa record transaksi berbeda berdasarkan canteen_id agar masing-masing Ibu Kantin hanya menerima daftar pesanan miliknya sendiri.
4

Penggunaan Voucher & Pembayaran E-Wallet TyU-Pay

Aksi Demo: Di halaman keranjang, coba masukkan kode voucher (contoh: buat promo di admin terlebih dahulu, atau gunakan voucher default). Pilih metode pembayaran **Saldo TyU-Pay**, lalu klik bayar.
Poin Penjelasan untuk Dosen:
  • Tunjukkan validasi voucher yang berjalan secara realtime menggunakan API endpoint. Diskon akan memotong total belanja secara instan.
  • Saat menggunakan Saldo TyU-Pay, sistem akan memotong saldo pembeli dan mentransfernya secara aman ke saldo penampungan gerai kantin masing-masing.
  • Demonstrasikan keamanan saldo: Jika saldo pembeli kurang dari total belanja, transaksi akan ditolak secara aman oleh backend (validasi sisi server di OrderController::store).
5

Tanya Jawab Pintar via AI Chatbot (Gemini API)

Aksi Demo: Klik icon robot asisten di sebelah tombol notifikasi di navbar. Tanyakan rekomendasi seperti: "saya mau makanan yang pedes tapi gak berkuah" atau "rekomendasi minuman dingin rasa buah dong".
Poin Penjelasan untuk Dosen:
  • Tunjukkan bagaimana chatbot memberikan rekomendasi hidangan dingin atau pedas kering secara akurat berdasarkan menu yang benar-benar dijual di kantin.
  • AI asisten merespons dengan logat kasual kampus Tel-U dan menyisipkan emoji, menghidupkan suasana pemesanan makanan.
6

Pemrosesan Pesanan oleh Ibu Kantin

Aksi Demo: Lakukan login sebagai akun Ibu Kantin (Bu Sari: busari_kantin / kantin123). Masuk ke halaman **Pesanan Masuk (Orders)**.
Poin Penjelasan untuk Dosen:
  • Tunjukkan bahwa pesanan dari langkah sebelumnya masuk ke daftar *Pending* secara instan.
  • Klik **Proses Masak** (status berubah menjadi *Processing*), lalu klik **Selesai** (status menjadi *Completed*).
  • Jelaskan bahwa pembeli akan menerima notifikasi status pesanan secara dinamis di dasbor mereka ketika Ibu Kantin memperbarui status.
  • Tunjukkan fitur ekspor data menu & pesanan ke PDF/CSV untuk pembukuan gerai.

Panel Kontrol Utama (Admin Panel)

Aksi Demo: Login sebagai Admin (admin / admin123). Jelaskan menu navigasi admin: Dashboard, Kelola Pengguna, Voucher, dan Ekspor Rekap.
Poin Penjelasan untuk Dosen:

  • Mendemonstrasikan pembuatan voucher diskon global baru, lengkap dengan batas kuota dan kedaluwarsa.

💬 Tanya Jawab (Q&A) yang Sering Diajukan Dosen

Siapkan jawaban-jawaban teknis berikut jika dosen memberikan pertanyaan mendalam saat sidang/demo:

❓ Bagaimana cara kerja pemisahan pesanan (Split Order) ketika pembeli membeli dari beberapa kantin sekaligus?
Jawaban Teknis:
Di dalam database, transaksi utama dicatat dalam tabel payments untuk mencatat total uang yang dibayar dan metode pembayarannya. Namun, untuk daftar belanjaan, backend (OrderController::store) akan mendeteksi variasi canteen_id dari item di keranjang. Sistem akan melakukan perulangan (*loop*) dan memecah item tersebut menjadi baris-baris baru di tabel orders terpisah untuk setiap kantin. Dengan cara ini, relasi data tetap terjaga, dan Ibu Kantin hanya dapat melihat pesanan yang didelegasikan ke kantinnya.
❓ Bagaimana Anda memastikan chatbot AI tidak merekomendasikan makanan fiktif yang tidak dijual di kantin?
Jawaban Teknis:
Di dalam ChatbotController::chat, sebelum mengirimkan pesan pengguna ke API Gemini, aplikasi melakukan query terlebih dahulu ke database: Menu::where('is_available', true)->with('canteen')->get(). Hasil query ini diubah menjadi teks daftar menu dan disisipkan ke dalam systemInstruction (konteks instruksi sistem) sebagai database referensi tunggal. AI diinstruksikan secara tegas untuk hanya memberikan rekomendasi berdasarkan daftar tersebut.
❓ Mengapa Anda menggunakan TailwindCSS v4.0.0 dibanding framework CSS konvensional?
Jawaban Teknis:
TailwindCSS v4.0.0 menggunakan compiler baru berbasis Rust yang terintegrasi langsung dengan Vite secara native melalui plugin @tailwindcss/vite. Hal ini membuat waktu build aset CSS menjadi 10x lebih cepat, menghasilkan ukuran file produksi yang sangat kecil (hanya menyertakan utility class yang benar-benar digunakan), serta mendukung fitur modern CSS seperti CSS Variables tingkat lanjut.
❓ Bagaimana alur penanganan status pembayaran online jika pembeli menutup popup sebelum bayar?
Jawaban Teknis:
Aplikasi menggunakan sistem **Midtrans Notification Webhook**. Ketika pembeli melakukan transaksi via Midtrans Snap, data disimpan dengan status pending. Jika pembeli membayar, atau menutup transaksi hingga kedaluwarsa, server Midtrans secara asynchronous akan mengirimkan request POST ke webhook endpoint aplikasi kita (/payment/webhook). Webhook Controller kemudian memproses payload tersebut dan memperbarui status di tabel payments dan orders secara otomatis di latar belakang.
Persiapan Sebelum Demo Berlangsung:

Pastikan Anda sudah menjalankan server lokal menggunakan perintah php artisan serve dan compiler npm run dev. Sangat disarankan untuk mengosongkan/mereset database sebelum demo dimulai agar statistik terlihat bersih dengan menjalankan perintah php artisan migrate:fresh --seed.
